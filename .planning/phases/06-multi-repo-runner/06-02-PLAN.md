<task type="auto">
  <name>Task 1: Implement the multi-repo loop in handle()</name>
  <files>app/Commands/RunCommand.php</files>
  <action>
    - Update `handle()` to implement the iteration logic per D-04, D-05.
    - Logic:
        1. Resolve `repo` argument.
        2. If `repo` provided: single run (use `runRepo` at `getcwd()`).
        3. If NO `repo` provided:
            a. Check `globalConfig->configuredRepos()`.
            b. If NOT empty: loop through each normalized repo.
            c. If EMPTY: fallback to single run at `getcwd()` (legacy current-dir behavior).
    - Inside the loop:
        - Call `chdir($path)` before `runRepo()`.
        - Catch `Throwable` around `runRepo()` call to ensure error isolation (D-08).
        - Collect `RunResult` for each repo.
        - Log starting/finished for each repo to distinguish them (D-11).
    - Ensure `RunProgressSnapshot` and usage reporting are updated properly for each repo.
  </action>
  <verify>
    <automated>./vendor/bin/pest tests/Feature/RunCommandTest.php</automated>
  </verify>
  <done>RunCommand correctly iterates multiple repos or fallbacks appropriately.</done>
