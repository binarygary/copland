  <name>Task 1: Create OpenAiCompatClient with bidirectional message translation</name>
  <files>

  </files>
  <behavior>
    complete() — message translation OUT (Anthropic → OpenAI):
    - SystemBlock[] with cache=true or cache=false → single ['role' => 'system', 'content' => implode("\n\n", texts)] prepended. Cache flag stripped (D-11).
    - role=user message where content is array with type=tool_result blocks → converted to separate role=tool messages (one per block): ['role' => 'tool', 'tool_call_id' => $block['tool_use_id'], 'content' => $block['content']]
    - role=assistant message where content is array with type=tool_use blocks → ['role' => 'assistant', 'content' => null, 'tool_calls' => [{id, type:'function', function:{name, arguments:json_encode(input)}}]]
    - role=assistant message with only text blocks → ['role' => 'assistant', 'content' => imploded text]
    - role=user plain string content → passed through unchanged

    complete() — response translation IN (OpenAI → LlmResponse):
    - $message->content (non-null string) → ['type' => 'text', 'text' => $content] block
    - $message->toolCalls (array) → ['type' => 'tool_use', 'id' => $tc->id, 'name' => $tc->function->name, 'input' => json_decode($tc->function->arguments, true) ?? []]
    - stopReason: $choice->finishReason ?? 'stop' (already canonical, no normalization needed)
    - LlmUsage: promptTokens → inputTokens, completionTokens → outputTokens, cacheWrite=0, cacheRead=0

    Tool schema translation:
    - Tools array passed through ToolSchemaTranslator::translateAll() before sending
    - Empty tools array → omit 'tools' key from request (do not send null or empty array)

    OpenRouter detection:
    - Constructor receives a 'provider' string ('ollama' or 'openrouter') or null
    - When provider === 'openrouter', HTTP-Referer and X-Title headers already baked into the underlying client by LlmClientFactory (D-19). No per-call header injection needed in complete().

    TOOL_CAPABLE_MODELS constant:
    - Public const array on OpenAiCompatClient (used by LlmClientFactory for warning check)
    - Contains models from the list in interfaces section above
  </behavior>

    - When provider is 'openrouter': build OpenAiCompatClient using OpenAI::factory()->withApiKey($config['api_key'])->withBaseUri('https://openrouter.ai/api/v1')->withHttpHeader('HTTP-Referer', 'https://github.com/binarygary/copland')->withHttpHeader('X-Title', 'Copland')->make()
    - LlmClientFactory::ollamaStageConfigs(GlobalConfig $global, ?RepoConfig $repo): returns array of ['base_url', 'model'] entries for all ollama-configured stages (used by RunCommand for probe/warning). Deduped by base_url.

    - Add three per-service bindings: ClaudeSelectorService, ClaudePlannerService, ClaudeExecutorService

    - Since RepoConfig requires a path (not available at container registration time), bind AnthropicApiClient as fallback default for container resolution; RunCommand continues to manually wire per its existing pattern (see critical note below)

    - RunCommand will be updated in Plan 03 to call LlmClientFactory::forStage() per stage with the real RepoConfig.
    - So AppServiceProvider change is: replace `$this->app->bind(LlmClient::class, AnthropicApiClient::class)` with a closure that calls `LlmClientFactory::forStage('executor', app(GlobalConfig::class))`.
